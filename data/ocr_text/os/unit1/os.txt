## Que 1.1. What is operating system? Write the major functions of an operating system.

### Answer

**Operating system:**

1. An operating system (OS) is a software that manages computer hardware and provides services for computer programs.
2. It acts as an intermediary between users and the computer's hardware.
3. The OS ensures proper functioning of applications, file management, and hardware coordination.

**Functions of an operating system:** The operating system performs the following functions:

1. **Booting:** The OS initializes the computer's hardware and loads the essential software components during the booting process, enabling the system to start and become operational.
2. **Formatting:** The OS allows users to format (initialize) diskettes so they can store data and programs.
3. **Managing computer resources:** It allocates and controls resources such as CPU time, memory, and peripherals to ensure efficient multitasking and optimal performance for running applications.
4. **Managing files:** The OS organizes and maintains files and enable users to create, modify, delete, and organize data in a systematic manner.
5. **Managing tasks:** The OS oversees the execution of various tasks and processes, prioritizing and scheduling them to ensure smooth operation.

## Que 1.2. Describe the classification of operating system.

**OR**

**Write down the different types of operating system.**

### Answer

Various types of operating system are as follows:

1. **Batch system:**
i. A batch operating system processes jobs in groups or batches without direct user interaction.
ii. Users submit jobs, and the system executes them sequentially.
iii. In this execution environment, the CPU is often idle.

## 2. Multi-programming system :

i. In this type of operating system, more than one program resides into main memory.
ii. The operating system picks and begins to execute one of the jobs in the memory.
iii. While one program is waiting for I/O, the CPU can execute another.
iv. As long as there is some job to execute, the CPU will never be idle.

## 3. Time-sharing system :

i. Time-sharing OS enables multiple users to share the computer simultaneously.
ii. It uses CPU scheduling, ensuring each user gets a fair share of computing resources.

## 4. Real time operating system (RTOS) :

i. RTOS is designed for applications where tasks must be completed within specific time constraints.
ii. In RTOS there are rigid time requirements on the operation of a processor or the flow of data.
iii. RTOS is suitable for embedded systems, control systems, and applications with critical deadlines.

**Que 1.4.** Discuss essential properties of time-sharing operating system, real time operating system and distributed operating system.

**Answer**

**A. Properties of time-sharing operating system :**

1. **Multitasking** : Enables multiple users to run programs concurrently.
2. **Interactive response** : Prioritizes quick response times to user inputs.
3. **Fair resource allocation** : Ensures equitable distribution of system resources among users.
4. **Time slicing** : Divides CPU time into small intervals.
5. **User interface** : Provides a user-friendly interface, allowing users to interact with the system in real-time.

**B. Properties of real time operating system :**

1. **Predictability** : Guarantees timely and deterministic responses to external events.
2. **Task scheduling** : Utilizes priority-based scheduling algorithms to ensure high-priority tasks are executed within their deadlines.
3. **Reliability** : Emphasizes system reliability and stability.
4. **Concurrency control** : Manages concurrent processes carefully.
5. **Minimal downtime** : Aims for minimal system downtime.

**C. Properties of distributed operating system :**

1. **Resource sharing** : Facilitates sharing of resources across a network of interconnected computers.
2. **Transparency** : Strives for transparency in accessing resources.
3. **Fault tolerance** : Incorporates mechanisms to handle and recover from failures.
4. **Concurrency control** : Implements distributed algorithms to manage concurrent access to shared resources.
5. **Communication** : Emphasizes efficient communication protocols between distributed components.

## Answer

B. Layered structure of an OS:

1. It is a design approach where the OS is organized into distinct layers, each providing specific functionalities and services.

![img-1.jpeg](img-1.jpeg)
Fig. 1.15.1.

1-12 D (CS/IT-Sem-4)
Introduction

2. The bottom layer (layer 0), is the hardware; the highest (layer N) is the user interface.

3. Each layer interacts with the layers directly above and below it, creating a hierarchical structure.

4. This design enhances modularity, scalability, and ease of maintenance.

5. Common layers in a layered operating system include:

i. Hardware layer:
a. The lowest layer interacts directly with the computer hardware, managing devices, and providing a uniform interface for the layers above.
b. It deals with issues like interrupt handling and memory management.

ii. Kernel layer:
a. The kernel layer is responsible for essential services like process management, memory management, file system access, and device drivers.
b. It acts as an intermediary between user applications and the hardware.

iii. System call interface layer:
a. This layer provides an interface for user programs to interact with the kernel.
b. System calls act as bridges between user-level applications and the kernel.

iv. User interface layer:
a. The user interface layer is responsible for presenting information to users and receiving their input.
b. It includes both the command-line interface (CLI) and graphical user interface (GUI).

C. Advantages of layered structure:

1. Modularity: Changes in one layer do not affect other layers, promoting ease of maintenance and updates.

2. Abstraction: Each layer hides the complexity of the layers below, allowing developers to focus on specific functionalities.

3. Portability: Layers can be implemented independently, making it easier to port the operating system to different hardware architectures.

4. Scalability: New layers or functionalities can be added without affecting existing layers, facilitating system expansion.

Operating System

## D. Disadvantages of layered structure :

1. Overhead : Communication between layers can introduce overhead, impacting system performance.
2. Rigidity : Fixed layer interactions may make it challenging to accommodate changes in hardware or system requirements.
3. Complexity in design : Defining clear interfaces between layers and ensuring proper communication can be complex.
4. Resource consumption : Some layers may consume resources even if not fully utilized, leading to inefficiencies in resource usage.

## Que 1.21. Describe re-entrant kernels.

### Answer

1. A re-entrant kernel is one that allows the same code or routine to be reentered and executed concurrently by multiple processes or threads without causing conflicts.

2. Reentrant kernels are designed to support concurrent execution of code in a multi-process or multi-threaded environment.

3. Reentrant kernels ensure thread safety by designing kernel code and data structures to be immune to race conditions.

4. Reentrant kernels avoid global data conflicts by using local or thread-specific storage for critical data.

5. Reentrant kernels are designed for scalability, efficient resource utilization, and improved system responsiveness in multi-process or multi-threaded environments.

6. Reentrant kernels are particularly beneficial in modern operating systems where parallelism and concurrency are prevalent.

7. They enable the efficient execution of multiple tasks simultaneously without compromising data integrity or system stability.

**Que 1.24.** What do you understand by system call ? How is a system call made ? How is a system call handled by the system ? Choose suitable examples for explanation.

**Answer**

A. System call :

1. A system call is a mechanism that enables a program to request a service from the operating system's kernel.
2. It provides a way for user-level applications to access functionalities or resources that are not directly accessible in user mode, such as I/O operations, process management, and file system operations.

B. How a system call is made : A system call is typically triggered through following specific instruction or interrupt that transfers control from user mode to kernel mode :

1. Software interrupt : User-level programs often use software interrupts (e.g., INT 0x80 in x86 architecture) to transition from user mode to kernel mode.
2. Special instruction : On some architectures, there are special instructions (e.g., syscall in x86_64) specifically designed for making system calls.
3. Library functions : High-level languages often provide library functions that encapsulate system calls, making them accessible through standard function calls.

1-20 D (CS/IT-Sem-4)
Introduction

C. How a system call is handled by the system: Once a system call is initiated, the operating system needs to handle the request. The sequence of events typically involves the following steps:

1. Context switch: The CPU switches from user mode to kernel mode, allowing the system call to execute privileged instructions.
2. System call number: The system call number is usually passed as a parameter to the kernel.
3. Parameters: Additional parameters required for the operation are passed to the kernel.
4. Validation: The kernel validates the parameters to ensure they are within permissible ranges and do not pose a security risk.
5. Execution: The kernel performs the requested operation on behalf of the user-level program.
6. Result: The result of the system call is communicated back to the user-level program.

D. Examples:

1. File I/O (open, read, write): A program might use the 'open' system call to open a file. The kernel handles the request by locating the file, checking permissions, and returning a file descriptor.
2. Process Creation (fork): The 'fork' system call is used to create a new process. The kernel duplicates the calling process, creating a new one with its own memory space.

## Que 3.1.

What is CPU scheduling? Give the criteria for scheduling.

### Answer

1. CPU scheduling is the management of CPU resources.
2. The scheduling mechanism is the part of the process manager that handles the removal of the running process from the CPU and the selection of another process.
3. CPU scheduling is the basis of multi-programmed operating systems.
4. The scheduler is responsible for multiplexing processes on the CPU.
5. CPU scheduling is used to increase CPU utilization.

### Scheduling criteria:

Some commonly used criteria are:

1. **CPU utilization**:
i. CPU utilization is the average function of time, during which the processor is busy.
ii. The load on the system affects the level of utilization that can be achieved.

2. **Throughput**:
i. Throughput refers to the amount of work completed in a unit of time.
ii. It is a measure of number of processes the system can execute in a period of time. The higher the number, the more work is done by the system.

3. **Waiting time**:
i. The average period of time a process spends waiting.
ii. Waiting time may be expressed as turnaround time less the actual execution time.

4. **Turnaround time**:
i. The interval from the time of submission of a process to the time of completion is the turnaround time.
ii. Turnaround time is the sum of the periods spent waiting to get into memory, waiting into the ready queue, executing on the CPU and doing I/O.

Operating System
3-3D (CS/IT-Sem-4)

5. Response time:
i. Response time is the time from the submission of a request until the first response is produced.

6. Priority:
i. Give preferential treatment to processes with higher priorities.

7. Balanced utilization:
i. Utilization of memory, I/O devices and other system resources are also considered.
ii. Not only CPU utilization considered for performance.

8. Fairness:
i. Avoid the process from the starvation.
ii. All the processes must be given equal opportunity to execute.

**Que 3.2.** Discuss the objectives of CPU scheduling.

**Answer**

**Objectives of CPU scheduling:**

1. Efficiency: Keep the CPU busy all the time.
2. Maximize throughput: Service the largest possible number of jobs in a given amount of time; minimize the amount of time users must wait for their results.
3. Minimize response time: Interactive users should see good performance.
4. Minimize overhead: Do not waste too many resources. Keep scheduling time and context switch time at a minimum.
5. Maximize resource use: Favour processes that will use under-utilized resources.
6. Avoid indefinite postponement: Every process should get a chance to run eventually.
7. Enforce priorities: If the scheduler allows a process to be assigned a priority, it should be meaningful and enforced.

**Que 3.18.** Discuss First Come First Serve (FCFS) scheduling algorithm. Give its advantages and disadvantages.

**Answer**

1. The simplest scheduling algorithm is the First Come First Served (FCFS) algorithm, *i.e.*, the process, which requests the CPU first is allocated the CPU first.

2. The FCFS algorithm is simply realized with a FIFO queue. It functions as follows :

a. When a process enters the ready queue, its Process Control Block (PCB) is linked onto the tail of the queue.

b. As soon as the CPU is free, it is allocated to the process at the head of the ready queue.

c. The running process is then removed from the ready queue.

**Advantages of First Come First Served (FCFS) algorithm :**

1. Better for long processes.

2. Simple method (*i.e.*, minimum overhead on processor).

3. No starvation.

**Disadvantages of First Come First Served (FCFS) algorithm :**

1. Convoy effect occurs. Even very small process should wait for its turn to come to utilize the CPU.

2. Short process behind long process results in lower CPU utilization.

3. Throughput is not emphasized.

**Que 3.19.** Discuss Shortest Job First (SJF) scheduling algorithm. Also, write its advantages and disadvantages.

Operating System
3-19 D (CS/IT-Sem-4)

# Answer

1. SJF algorithm associates with each process the length of the process's next CPU burst.
2. When the CPU is available, it is assigned to the process that has the smallest next CPU burst.
3. If the next CPU bursts of two processes are the same, FCFS scheduling is used to break the tie.
4. The SJF algorithm can be either preemptive or non-preemptive.

## Advantages of Shortest Job First (SJF) algorithm:

1. It gives superior turnaround time performance to shortest process next because a short job is given immediate preference to a running longer job.
2. Throughput is high.

## Disadvantages of Shortest Job First (SJF) algorithm:

1. Elapsed time (i.e., execution-completed-time) must be recorded, it results an additional overhead on the processor.
2. Starvation may be possible for the longer processes.

# Answer

1. Priority scheduling is a non pre-emptive algorithm and one of the most common scheduling algorithms in batch systems.
2. Each process is assigned a priority. Process with highest priority is to be executed first and so on.
3. Processes with same priority are executed on FCFS basis.
4. Priority can be decided based on memory requirements, time requirements or any other resource requirements.

## Advantages of priority scheduling algorithm:

1. The priority of a process can be selected based on memory requirement, time requirement or user preference.

## Disadvantages of priority scheduling algorithm:

1. If high priority processes use up a lot of CPU time, lower priority processes may starve and be postponed indefinitely.
2. Another problem with priority scheduling is deciding which process gets which priority level assigned to it.

**Que 3.21.** Discuss Round Robin scheduling algorithm. Give its advantages and disadvantages.

**Answer**

1. The Round Robin (RR) scheduling algorithm is designed especially for time sharing systems.
2. It is similar to FCFS scheduling, but pre-emption is added to enable the system to switch between processes.
3. The ready queue is treated as a circular queue.
4. The CPU scheduler goes around the ready queue, allocating the CPU to each process for a time interval of up to 1 time quantum.

**Advantages of Round Robin (RR) algorithm :**

1. Round-Robin is effective in a general-purpose, time sharing system or transaction processing system.
2. Fair treatment for all the processes.
3. Overhead on processor is low.

**Disadvantages of Round Robin (RR) algorithm :**

1. Care must be taken in choosing quantum value.
2. Processing overhead is there in handling clock interrupt.
3. Throughput is low if time quantum is too small.

**Que 3.29.** What is deadlock? What are the necessary conditions for deadlock?

**Answer**

**A. Deadlock :**

1. A deadlock is a situation in computing where two or more processes are unable to proceed because each is waiting for the other to release a resource.

2. In other words, a deadlock occurs when a set of processes are blocked because each process is holding a resource and waiting for another resource acquired by some other process.

**B. Necessary conditions for deadlock :** Necessary (Coffman) conditions for deadlock are as follows :

1. **Mutual exclusion :** At least one resource must be held in a non-shareable mode, meaning only one process can use it at a time.

Operating System
3-29 D (CS/IT-Sem-4)

2. **Hold and wait**: A process must be holding at least one resource and waiting to acquire additional resources that are currently held by other processes.

3. **No pre-emption**: Resources cannot be forcibly taken away from a process. They must be released voluntarily by the process holding them.

4. **Circular wait**: There must exist a circular chain of two or more processes, each waiting for a resource held by the next process in the chain.

C. **Example**: To illustrate these conditions, consider a simple example with two processes (P1 and P2) and two resources (R1 and R2):

1. **Mutual exclusion**: Both processes need exclusive access to resources. For example, P1 holds R1, and P2 holds R2.

2. **Hold and wait**: Suppose P1 holds R1 and requests R2, while P2 holds R2 and requests R1. Both processes are holding a resource and waiting for the other resource.

3. **No pre-emption**: Resources cannot be pre-empted. If P1 holds R1, it cannot be forcefully taken away; P1 must voluntarily release it.

4. **Circular wait**: There is a circular chain of waiting. P1 is waiting for R2 held by P2, and P2 is waiting for R1 held by P1, creating a circular dependency.

Request$_i \leq$ Work

Finish[ i ] = true
Go to step 2

**Step 5:** If $[i] =$ false, for some $i$, $1 \leq i \leq n$ then system is in deadlock state.

**Que 3.47. Discuss the techniques to recover from deadlock.**

**Answer**

There are two options for recovering from a deadlock:

1. **Process termination**: To eliminate deadlocks by aborting a process, we use one of two methods.

a. **Abort all deadlocked processes**: This method clearly will break the deadlock cycle, but the deadlocked processes may have been computed for a long time, and the results of these partial computations must be discarded and probably will have to be recomputed later.

b. **Abort one process at a time until the deadlock cycle is eliminated**: This method incurs considerable overhead, since after each process is aborted, a deadlock-detection algorithm must be invoked to determine whether any processes are still deadlocked.

2. **Resource pre-emption**: To eliminate deadlocks using resource pre-emption, we successively preempt some resources from processes and give these resources to other processes until the deadlock cycle is broken.

a. **Selecting a victim**:

i. Which resources and which processes are to be preempted? As in process termination, we must determine the order of pre-emption to minimize cost.

**Que 4.3.** What is fixed size memory partitioning? Also give its advantages and disadvantages.

**Answer**

**Fixed partitioning :**
1. Fixed-size memory partitioning is a memory management technique used in operating systems to allocate and manage memory for processes.
2. In this scheme, the physical memory is divided into fixed-sized partitions, and each partition can accommodate exactly one process.
3. The size of each partition remains constant throughout the execution of the program.
4. This memory management technique was commonly used in early computer systems.
5. However, due to its limitations, it has been replaced by dynamic partitioning and paging in modern operating systems.

**Advantages of fixed-size memory partitioning :**
1. **Simplicity** : Fixed-size memory partitioning is straightforward to implement and manage.
2. **Low overhead** : The overhead associated with managing fixed-size partitions is relatively low.
3. **Ease of implementation** : Fixed-size partitioning is easier to implement.

**Disadvantages of fixed-size memory partitioning :**
1. **Internal fragmentation** : It is one of the major drawbacks, where the allocated memory partition may be larger than the actual memory needs of the process.
2. **Inflexibility** : Fixed-size partitions are not flexible in accommodating processes of varying sizes.
3. **Poor memory utilization** : Due to internal fragmentation and inflexibility, the overall memory utilization is often inefficient.

**Que 4.6.** Describe memory allocation.

4-6D (CS/IT-Sem-4)
Memory Management

# Answer

1. Memory allocation refers to the process of assigning portions of the computer's memory to various programs and processes.
2. One of the simplest methods for memory allocation is to divide memory into several fixed-sized partitions.
3. When a partition is free a process is selected from the input queue and is loaded into the free partition.
4. When the process terminates, the partition becomes available for another process.
5. The operating system keeps a table indicating which parts of memory are available and which are occupied.
6. Initially, all memory is available for user processes, and is considered as one large block of available memory, a hole.
7. When a process arrives and needs memory, we search for a hole large enough for this process.
8. If we find one, we allocate only as much memory as is needed.
9. When a process is allocated space, it is loaded into memory and it can then compete for the CPU.
10. When a process terminates, it releases its memory, which the operating system may then fill with another process from the input queue.

## Que 4.7. Discuss dynamic memory allocations techniques.

OR

Discuss first fit, best fit and worst fit memory allocation techniques.

# Answer

Dynamic memory allocation techniques :

1. First fit: Allocate the first hole that is big enough. Searching can start either at the beginning of the set of holes or at the location where the previous first fit search ended. We can stop searching as soon as we find a free hole that is large enough.
2. Best fit: Allocate the smallest hole that is big enough. We must search the entire list, unless the list is ordered by size. This strategy produces the smallest leftover hole.
3. Worst fit: Allocate the largest hole. Again, we must search the entire list, unless it is sorted by size. This strategy produces the largest leftover hole, which may be more useful than the smaller leftover hole from a best fit approach.

**Que 4.10.** What are the different techniques to remove fragmentation in case of multiprogramming with fixed partitions and variable partitions?

**Answer**

**A. Fragmentation removal in case of multiprogramming with fixed partitions:**

1. **Compaction:**
i. Compaction involves rearranging the allocated and free memory blocks to create larger contiguous free memory space.
ii. This technique is applicable in situations where external fragmentation occurs.
iii. By moving processes to consolidate free memory, compaction can reduce or eliminate external fragmentation.

2. **Dynamic partitioning:**
i. Dynamic partitioning allows the operating system to dynamically allocate memory to processes based on their actual requirements.
ii. This approach helps mitigate internal fragmentation by allocating only the amount of memory needed by each process thereby reducing wasted space within partitions.

**B. Fragmentation removal in case of multiprogramming with variable partitions:**

1. **Paging:**
i. Memory is divided into fixed-size blocks called pages, and processes are allocated in these page-sized units.
ii. This helps in reducing external fragmentation.

2. **Segmentation:**
i. Memory is divided into variable-sized logical segments.
ii. Segmentation helps manage variable-sized memory requirements more flexibly, reducing both internal and external fragmentation.

3. **Best fit, First fit, and worst fit algorithms:**
i. These algorithms are used for dynamic memory allocation.
ii. Best fit allocates the smallest block that fits the requirement, first fit allocates the first block that is large enough, and worst fit allocates the largest available block.

**Que 4.12.** What do you mean by paging ? How page table can be implemented ?

**Paging :**

**Que 4.34.** Discuss the LRU page replacement policy. Prove that this policy do not suffer from Belady’s anomaly. How many page faults would occur with LRU policy for the following reference string:

**Answer**

LRU page replacement: Refer Q. 4.28, Page 4–25D, Unit-4.

**Proof:**

1. Let’s say, initially, the system has ‘k’ page frames, and there are ‘n’ pages in the sequence.

2. When the number of frames is less than ‘n’, every page access will cause a page fault since not all pages can be accommodated in the available frames.

3. Once the number of frames becomes equal to ‘n’ or greater, LRU will have all pages in memory, and there will be no page faults.

4. Increasing the number of frames beyond ‘n’ does not cause additional page faults because, in the worst case, each page will have its own frame, and the LRU policy ensures that the least recently used page is replaced when needed.

5. Therefore, for a sequence of ‘n’ pages, increasing the number of frames beyond ‘n’ in the LRU policy does not introduce additional page faults.

6. This property ensures that LRU is free from Belady’s anomaly.

## Answer

**A. Thrashing:**

1. Thrashing occurs when a system spends more time in processing page faults rather than executing transactions.
2. As the page fault rate increases, more transactions need processing from the paging device.

Fig. 4.35.1.

3. The queue at the paging device increases, resulting in increased service time for a page fault.

4-32 D (CS/TT-Sem-4)
Memory Management

5. Thrashing becomes a greater threat as the degree of multiprogramming of the system increases.
6. Therefore, to increase CPU utilization multiprogramming should be reduced.

B. Causes of thrashing: The key factors contributing to thrashing include:
1. Overloading: The system is trying to run too many processes concurrently.
2. Inadequate page frame allocation: The number of allocated page frames may be insufficient to hold the working sets of active processes.
3. Poor locality of reference: If processes exhibit poor spatial or temporal locality of reference it can lead to excessive page faults and thrashing.

C. Detection of thrashing: Following methods can be used to identify thrashing:
1. Page fault rate monitoring: If the page fault rate is consistently high and the CPU utilization is low, it may indicate thrashing.

3. Utilization metrics: Tracking metrics such as CPU utilization and disk I/O can help identify thrashing.

D. Preventing or mitigating thrashing:
1. Optimizing page replacement policies: Choosing effective page replacement algorithms, such as LRU or LFU, can help minimize thrashing.
2. Increasing physical memory: Adding more RAM to the system can alleviate thrashing by providing additional space.
3. Adjusting process priorities: Adjusting the priorities of processes can help allocate resources more efficiently.

**Que 4.38.** What is meant by cache mapping? What are different types of mapping? Discuss different mapping techniques with examples.

# Answer

**Cache mapping**:
1. Cache mapping is the method by which the contents of main memory are brought into the cache and referenced by the CPU.
2. Mapping is a process to discuss possible methods for specifying where memory blocks are placed in the cache. Mapping function dictates how the cache is organized.

**Types of mapping**:
i. **Direct mapping**:
1. The direct mapping technique is simple and inexpensive to implement.
2. When the CPU wants to access data from memory, it places an address. The index field of CPU address is used to access address.
3. The tag field of CPU address is compared with the associated tag in the word read from the cache.
4. If the tag-bits of CPU address are matched with the tag-bits of cache, then there is a hit and the required data word is read from cache.

4-34 D (CS/IT-Sem-4)
Memory Management

5. If there is no match, then there is a miss and the required data word is stored in main memory. It is then transferred from main memory to cache memory with the new tag.

Example of direct mapping: If a cache has 8 blocks and the main memory has 32 blocks, each main memory block is mapped to a specific cache block (e.g., using modulo 8). This creates a one-to-one relationship between main memory and cache blocks.

ii. Associative mapping:
1. An associative mapping uses an associative memory.
2. This memory is being accessed using its contents.
3. Each line of cache memory will accommodate the address (main memory) and the contents of that address from the main memory.
4. That is why this memory is also called Content Addressable Memory (CAM). It allows each block of main memory to be stored in the cache.

Example associative mapping: If a cache has 8 blocks, any of the 32 blocks in main memory can be placed in any cache block. This flexibility comes at the cost of increased hardware complexity.

iii. Set associative mapping:
1. That is the easy control of the direct mapping cache and the more flexible mapping of the fully associative cache.
2. In set associative mapping, each cache location can have more than one pair of tag + data items.
3. That is more than one pair of tag and data are residing at the same location of cache memory. If one cache location is holding two pair of tag + data items, that is called 2-way set associative mapping.

Example of set associative mapping: If the cache is 4-way set-associative, each set contains four blocks. A main memory block can be placed in any block within its corresponding set.

**Que 5.15.** Explain the following methods:
i. Bit vector
ii. Linked list
iii. Grouping
iv. Counting

OR

Discuss free space management in disk.

**Answer**

i. Bit vector:
1. Frequently, the free-space list is implemented as a bit map or bit vector.
2. Each block is represented by 1 bit.
3. If the block is free, the bit is 1; if the block is allocated, the bit is 0.
4. The main advantage of this approach is its relative simplicity and its efficiency in finding the first free block or $n$ consecutive free blocks on the disk.

ii. Linked list:
1. In this approach, the free disk blocks are linked together.
2. The block number of the very first disk block is stored at a separate location on disk and is also cached in memory.
3. The first block contains a pointer to the next free disk block, and so on.
4. This scheme is not efficient because to traverse the list, we must read each block, which requires substantial I/O time.

5-18 D (CS/IT-Sem-4) I/O Management and Disk Scheduling

![img-52.jpeg](img-52.jpeg)
Fig. 5.15.1. Linked free-space list on disk.

# iii. Grouping :

1. A modification of the free list approach stores the addresses of  $n$  free blocks in the first free block.
2. The first  $n - 1$  of these blocks is actually free.
3. The last block contains the addresses of another  $n$  free block, and so on.
4. The addresses of a large number of free blocks can now be found quickly, unlike the situation when the standard linked list approach is used.

# iv. Counting :

1. This approach stores the address of the first free disk block and a number  $n$  of free contiguous disk blocks that follow the first block.
2. Every entry in the list would contain :

a. Address of first free disk block.
b. A number  $n$ .

3. Each entry in the free-space list then consists of a disk address and a count.
4. These entries can be stored in a balanced tree for efficient lookup, insertion, and deletion.

# Answer

## RAID:

1. RAID or redundant array of independent disks is a data storage virtualization technology that combines multiple physical disk drive components into one or more logical units for data redundancy, performance improvement, or both.
2. It is a way of storing the same data in different places on multiple hard disks or solid-state drives to protect data in the case of a drive failure.

## Characteristics:

1. Increased data storage capacity.
2. Improved data read and write performance.
3. Fault tolerance to handle disk failures.

## Different RAID Levels:

### A. RAID-0 (Stripping): RAID 0 splits data evenly across two or more disks, without parity information, redundancy, or fault tolerance.

#### Advantages:

1. It is easy to implement.
2. It utilizes the storage capacity in a better way.

#### Disadvantages:

1. A single drive loss can result in the complete failure of the system.
2. Not a good choice for a critical system.

### B. RAID-1 (Mirroring): RAID 1 consists of an exact copy of a set of data on two or more disks.

#### Advantages:

1. It covers complete redundancy.
2. It can increase data security and speed.

#### Disadvantages:

1. It is highly expensive.
2. Storage capacity is less.

### C. RAID-2 (Bit-Level Stripping with Dedicated Parity): RAID 2, stripes data at the bit (rather than block) level, and uses a Hamming code for error correction.

#### Advantages:

1. In case of error correction, it uses hamming code.
2. It uses one designated drive to store parity.

#### Disadvantages:

1. It has a complex structure and high cost due to extra drive.
2. It requires an extra drive for error detection.

5-20 D (CS/IT-Sem-4) I/O Management and Disk Scheduling

D. RAID-3 (Byte-Level Stripping with Dedicated Parity): RAID 3 consists of byte-level stripping with a dedicated parity disk.

**Advantages:**
1. Data can be transferred in bulk.
2. Data can be accessed in parallel.

**Disadvantages:**
1. It requires an additional drive for parity.
2. In the case of small-size files, it performs slowly.

E. RAID-4 (Block-Level Stripping with Dedicated Parity): RAID 4 consists of block-level stripping with a dedicated parity disk. As a result of its layout, RAID 4 provides good performance of random reads, while the performance of random writes is low due to the need to write all parity data to a single disk.

**Advantages:**
1. It helps in reconstructing the data if at most one data is lost.

**Disadvantages:**
1. It can't help in reconstructing when more than one data is lost.

F. RAID-5 (Block-Level Stripping with Distributed Parity): RAID 5 consists of block-level striping with distributed parity. Unlike in RAID 4, parity information is distributed among the drives. Upon failure of a single drive, subsequent reads can be calculated from the distributed parity such that no data is lost.

**Advantages:**
1. Data can be reconstructed using parity bits.
2. It makes the performance better.

**Disadvantages:**
1. Its technology is complex and extra space is required.
2. If both disks get damaged, data will be lost forever.

G. RAID-6 (Block-Level Stripping with two Parity Bits): RAID 6 extends RAID 5 by adding another parity block; thus, it uses block-level striping with two parity blocks distributed across all member disks.

**Advantages:**
1. Very high data Accessibility.
2. Fast read data transactions.

**Disadvantages:**
1. Due to double parity, it has slow write data transactions.
2. Extra space is required.

4. Each block contains a pointer to the next block.
5. These pointers are not made available to the user.
6. To create a new file, we simply create a new entry in the directory. With linked allocation, each directory entry has a pointer to the first disk block of the file.

## Advantages of linked allocation:

1. There is no external fragmentation. Only internal fragmentation is possible in the last block.
2. There is no need to declare size of the file at the time of its creation.

## Disadvantages of linked allocation:

1. Space is wasted in storing pointers.
2. Access time, seek time and latency time are higher than the contiguous allocation.

## iii. Indexed allocation:

1. Indexed allocation solves the problems caused by linked allocation by bringing all the pointers together into one location: the index block.
2. Each file has its own index block, which is an array of disk-block addresses.
3. The $i^{\text{th}}$ entry in the index block points to the $i^{\text{th}}$ block of the file.

4. The directory contains the address of the index block. To find and read the $i^{\text{th}}$ block, we use the pointer in the $i^{\text{th}}$ index-block entry.

5. When the file is created, all pointers in the index block are set to null.

6. When the $i^{\text{th}}$ block is first written, a block is obtained from the free-space manager, and its address is put in the $i^{\text{th}}$ index block entry.

**Advantages of index allocation :**

1. The entire block is available for data using this method.

2. There is no external fragmentation using this method.

**Disadvantage of index allocation :**

1. The entire index table must be in memory all the time to make it work.

2. Large space overhead (index).

What is access matrix? How it can be represented?

**Answer**

1. General model of protection can be viewed abstractly as a matrix, called an access matrix.
2. The rows of the access matrix represent domains, and the columns represent objects.
3. Each entry in the matrix consists of a set of access rights.
4. Because the column defines objects explicitly, we can omit the object name from the access right.
5. The entry access $(i,j)$ defines the set of operations that a process executing in domain $D_{i}$ can invoke on object $O_{j}$.
6. A protection system consists of mechanisms to control user access to system resources or to control information flow in the system.
7. The access matrix model consists of the following three components:
a. **Current objects**: Current object are a finite set of entities to which access is to be controlled. The set is denoted by 'O'. A typical example of an object is a file.
b. **Current subjects**: Current subjects are a finite set of entities that access current objects. The set is denoted by 'S'. A typical example of a subject is a process.
c. **Generic rights**: A finite set of generic rights, $R = \{r_1, r_2, r_3, \ldots, r_m\}$, gives various access rights that subjects can have to objects. Typical examples of such rights are read, write, execute, own, delete etc.

2. Overhead: Paging introduces overhead in terms of maintaining page tables, handling page faults, and managing TLB.

## 4.9. What are advantages of segmentation?

### Advantages of segmentation:

1. No page fault: Whole segment is present at once.
2. No internal fragmentation: Segment size is customized to each process.

## 4.10. What are demerits of segmentation?

### Demerits of segmentation:

1. Considerable compaction overhead is incurred in order to support dynamic segment growth and eliminate fragmentation.
2. The maximum size of a segment is limited by the size of main memory.

## 4.11. What is demand paging?

Demand paging is the process of loading the page into memory on demand when page fault occurs.
